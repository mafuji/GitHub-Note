GitHub リリース後にタグを更新（上書き）する手順書
==================================================

■ 前提
- 既に GitHub 上にリリース（Release）とタグ（Tag）が存在している。
- 軽微な修正を行い、同じタグ名でリリースを更新したい。
- GitHub は「タグの上書き」を GUI ではサポートしていないため、ローカルでタグを強制更新し、リモートへ強制 push する必要がある。

--------------------------------------------------
1. ローカルの修正をコミットする
--------------------------------------------------
例:
    git add .
    git commit -m "Fix: minor update"

--------------------------------------------------
2. 既存のタグをローカルで削除する
--------------------------------------------------
    git tag -d v1.0.0

※ v1.0.0 は更新したいタグ名に置き換える。

--------------------------------------------------
3. 同じタグ名で新しいタグを作成する
--------------------------------------------------
    git tag v1.0.0

※ 必要なら annotated tag にする:
    git tag -a v1.0.0 -m "Update release"

--------------------------------------------------
4. リモートのタグを強制的に上書き push する
--------------------------------------------------
    git push origin v1.0.0 --force

※ --force を付けないと GitHub 上のタグは更新されない。

--------------------------------------------------
5. GitHub の Release ページを更新する
--------------------------------------------------
- GitHub の Web UI で対象の Release を開く。
- "Edit" を押して内容を更新する。
- "Publish release" を押して保存する。

※ タグは既に更新されているため、Release の内容だけ整えればよい。

--------------------------------------------------
6. 注意点
--------------------------------------------------
- タグの強制 push は履歴を書き換えるため、他の開発者が同じタグを参照している場合は注意。
- 公開済みのタグを上書きするのは一般的には推奨されないが、個人開発や軽微修正ではよく行われる。
- どうしても安全に行いたい場合は、新しいタグ（例: v1.0.1）を作る方が無難。
